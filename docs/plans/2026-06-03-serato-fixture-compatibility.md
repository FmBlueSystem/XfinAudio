# Serato Fixture Compatibility Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add fixture-based validation for Serato crate artifacts so generated `.crate` bytes can be checked against documented TLV expectations without touching live Serato libraries.

**Architecture:** Extend `serato_crate.py` with a minimal read-only parser/validator for the supported crate subset (`vrsn`, `otrk`, `ptrk`). Use deterministic binary fixtures in tests. Keep write safety unchanged.

**Tech Stack:** Python 3.11+, pytest, pydantic, ruff, uv.

---

## Non-goals

- No live Serato library writes.
- No Serato database V2 parsing or mutation.
- No unsupported crate tags beyond read-only unknown-tag reporting.
- No audio file reads or writes.

## Task 1: Parse supported Serato crate subset

**Files:**
- Modify: `src/xfinaudio/exporting/serato_crate.py`
- Test: `tests/test_serato_crate.py`

**Steps:**
1. Write failing tests for `parse_serato_crate_bytes(crate_bytes)`:
   - returns version string;
   - returns ordered `ptrk` paths;
   - handles repeated `otrk` records;
   - rejects truncated TLV records with a typed validation error.
2. Run focused tests and confirm RED.
3. Implement `SeratoCrateParseError` and `ParsedSeratoCrate` pydantic model.
4. Implement a bounded TLV reader:
   - 4-byte ASCII tag;
   - 4-byte big-endian payload length;
   - payload must fit remaining bytes;
   - decode `vrsn` and `ptrk` as UTF-16BE;
   - parse nested `otrk` payloads looking for `ptrk`.
5. Keep unknown top-level or nested tags ignored but counted/listed for diagnostics.
6. Run focused tests and full suite.
